## 2024-05-24 - N+1 Query Fix for Nested Medication Adherence Calculation
**Learning:** In the `student_directory` API view, checking a student's active medications and computing their historical adherence rate previously triggered duplicate nested queries per student during pagination. Simply adding `medications__logs` to `prefetch_related` broke the active medications check (`student.medications.filter(is_active=True)`) if we overwrote the default relation.
**Action:** When an endpoint needs both filtered records (e.g., `active_meds`) and complete historical records (e.g., calculating total `adherence_rate` across *all* of a student's medications), use Django's `Prefetch` object with `to_attr` to separate the historical records into a new Python attribute (e.g., `to_attr='all_medications_with_logs'`), then perform counts and sums Python-side to avoid creating new query overhead.
