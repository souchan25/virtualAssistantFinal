## 2024-03-04 - [Django Prefetching with Custom Attributes]
**Learning:** Using `Prefetch` with `to_attr` replaces standard querysets with Python lists. If you then nest another `Prefetch` on a child property without explicitly nesting the `Prefetch` objects, it fails to bind properly. Furthermore, in Django REST Framework serializers (`SerializerMethodField`), methods like `obj.logs.count()` will always trigger a new DB query even if the parent was prefetched, unless you explicitly check for your custom `to_attr` (e.g., `hasattr(obj, 'all_logs')`) and manually compute the result using the Python list (e.g., `len([log for log in obj.all_logs if log.status != 'pending'])`).
**Action:** When creating custom `to_attr` prefetches on deeply nested objects, chain them properly using the `queryset` property in `Prefetch()`, and update `SerializerMethodField` logic to manually check and iterate those Python objects instead of relying on the ORM query methods.
